# Gopher

A personal repository documenting my journey learning Go through the excellent Go by Example tutorial.

## Overview

As a engineer, I believe learning a new programming language requires more than simply reading documentation or watching tutorials. While the examples in Go by Example are intentionally concise, rewriting each one from scratch helps build familiarity with the language’s syntax, idioms, and tooling.

This repository serves as a collection of those implementations as I work through the tutorial and expand the set of programming languages I’m comfortable using professionally.

## Why This Repository Exists

When learning a new language, it’s easy to understand an example conceptually and move on. However, real fluency comes from repetition and practice.

The goal of this repository is to:

* Develop muscle memory for Go’s syntax and core language features
* Become comfortable with Go’s tooling and project structure
* Reinforce concepts through hands-on implementation rather than passive reading
* Build a foundation for future Go projects and backend development
* Document my progress as I learn the language

## Learning Approach

For each lesson in the Go by Example tutorial, I:

1. Read and understand the concept being introduced
2. Recreate the example myself
3. Run and experiment with the code
4. Modify examples when appropriate to deepen understanding

Although many of the examples are small, the focus is on consistent practice and repetition—an approach that has proven effective when learning new technologies.

## Topics Covered

This repository follows the progression of the Go by Example curriculum, including topics such as:

* Variables and constants
* Control flow
* Functions
* Structs and interfaces
* Goroutines and channels
* Error handling
* Collections and maps
* Concurrency patterns
* And more
